BASE TEÓRICA

Pilares de la Seguridad Informática (CIA)

Los tres principios fundamentales que deben protegerse en cualquier sistema

C

Confidencialidad

Solo personas autorizadas pueden acceder a la información. Se protege con cifrado, control de accesos y autenticación.

I

Integridad

La información no fue modificada sin autorización. Se protege con hashes, firmas digitales y control de cambios.

D

Disponibilidad

El sistema está accesible cuando se necesita. Se protege con redundancia, backups y planes de contingencia.

Tipos de Ataques a Conocer

💀

Zero-Day (Día Cero)

Explota una vulnerabilidad desconocida para la que aún no existe parche. El más peligroso porque no hay defensa previa.

🔓

Fuerza Bruta

Prueba sistemática de contraseñas o combinaciones hasta encontrar la correcta. Se mitiga con límite de intentos y 2FA.

🕵️

Man-in-the-Middle

El atacante se interpone entre dos partes que se comunican, pudiendo leer o modificar los mensajes sin ser detectado.

💣

Exploit / RCE

Código que explota un fallo para ejecutar comandos remotamente en el sistema víctima sin credenciales.

🧹

Ataque Wiper

Diseñado para eliminar datos permanentemente. Poco común pero muy destructivo. Diferente del ransomware.

🔑

Robo de Credenciales

Obtención de claves, tokens o certificados para acceder a sistemas incluso después de aplicado un parche.

CASO 1

Ciberataque a Servidores Microsoft SharePoint

Julio 2025 · Infraestructuras críticas de EE.UU. y Europa comprometidas

CVE-2025-53770 — Vulnerabilidad crítica en SharePoint Server (on-premise). Permite ejecución remota de código sin credenciales, explotando fallos en autenticación y deserialización.
Q1 ¿Qué tipo de ataque sufrieron los servidores SharePoint? ▶

ZERO-DAY Ataque de "día cero" mediante la vulnerabilidad CVE-2025-53770.

Cadena de exploits no autenticados que atacan servidores locales SharePoint. Aprovecha fallos en autenticación y deserialización → ejecución remota de código (RCE) completa sin credenciales.

⚠ Palabra clave: Exploit no autenticado + Zero-Day + RCE

Q2 ¿Afectó la Disponibilidad del Servicio? ▶

SÍ · CRÍTICO Los atacantes "secuestraron" repositorios de documentos públicos. Las agencias perdieron acceso total al material.

Además robaron claves criptográficas para reingresar incluso tras aplicar parches.

Q3 ¿Qué pilares CIA fueron afectados? ▶

CONFIDENCIALIDAD Robo de datos, correos, contraseñas y claves criptográficas de funcionarios.

INTEGRIDAD Ejecución remota de código sin credenciales comprometió la integridad de los sistemas.

DISPONIBILIDAD Repositorios secuestrados, organizaciones sin acceso a su información.

⚠ Los TRES pilares fueron afectados.

Q4 ¿Por qué quedaron vulnerables los servidores? ▶

1. Vulnerabilidad zero-day — desconocida, sin parche disponible.

2. Fallos en autenticación y deserialización de SharePoint.

3. Servidores locales conectados a internet y vinculados a Outlook y Teams (superficie de ataque amplia).

Q5 ¿Hubo equipo similar a Hacking Ético que detectó los accesos? ▶

SÍ Actuaron como equipos de detección similares al hacking ético:

Eye Security (Países Bajos)

Detectó y reportó más de 50 violaciones.

Palo Alto Networks

Identificó intentos de explotar miles de servidores.

CIS (Centro Seguridad Internet)

Notificó a ~100 organizaciones sobre su vulnerabilidad.

Q6 ¿Qué medidas tomó Microsoft para mitigar el ataque? ▶

✦ Envío de alertas a clientes sobre la vulnerabilidad detectada.

✦ Recomendó modificar configuración de SharePoint Server o desconectarlo de internet.

✦ Anunció cese del uso de ingenieros con base en China para soporte del Departamento de Defensa.

✦ Coordinación con CISA y FBI para respuesta al incidente.

Q7 ¿El ataque fue a nivel Red (Nube) o servidores locales? ▶

SERVIDORES LOCALES (ON-PREMISE)

El ataque afectó exclusivamente servidores locales conectados a internet. No afectó la nube (Microsoft 365 no fue comprometido).

⚠ Diferencia clave: local ≠ nube

Q8 ¿Las medidas de Microsoft garantizaban que nadie accedería a archivos cifrados? ▶

NO No garantizaban seguridad total.

Los atacantes ya habían robado claves criptográficas durante el ataque, lo que les permitiría reingresar incluso después del parche.

Frase clave: "Lanzar un parche el lunes o martes no ayuda a quienes fueron comprometidos en las últimas 72 horas"

CASO 3

Hackers Rusos · Espionaje Digital

Interceptación de comunicaciones mediante técnicas avanzadas

Q1 ¿Cómo llevaban a cabo el espionaje y a través de qué método? ▶

Los hackers rusos realizaban espionaje mediante interceptación de comunicaciones. Se posicionaban entre las partes comunicantes para capturar el tráfico de red sin ser detectados.

Al comprometer servidores vinculados a Outlook y Teams, podían monitorear comunicaciones internas, robar contraseñas, correos y datos sensibles de funcionarios.

Paso 1

Explotar vulnerabilidad en servidor SharePoint

Paso 2

Obtener acceso a servicios vinculados (Outlook, Teams)

Paso 3

Interceptar y robar comunicaciones de ejecutivos y funcionarios

Q2 ¿Cómo se llama el método para vulnerar las comunicaciones? ▶

MAN-IN-THE-MIDDLE (MitM) — "Hombre en el Medio"

El atacante se inserta en el canal de comunicación entre dos partes sin que ninguna lo sepa. Puede:

🔍 Leer los mensajes · ✏️ Modificar el contenido · 📤 Inyectar mensajes falsos

⚠ Término exacto: Man-in-the-Middle / MitM / Ataque de intermediario

Q3 Si pudieron interceptar comunicaciones, ¿qué medida de seguridad falló? ▶

Fallaron dos medidas clave:

FALLO 1 Cifrado de extremo a extremo (E2E) — Si los mensajes hubieran estado correctamente cifrados, el interceptor no podría leer su contenido aunque los capture.

FALLO 2 Autenticación mutua — Debería verificar la identidad de ambas partes para evitar que un tercero se interponga. Su ausencia habilitó el MitM.

El error de programación de Microsoft en 2023 que permitió hackeo de correos gubernamentales es ejemplo directo de este fallo.

Glosario de Términos

Conceptos técnicos clave del trabajo práctico

ZERO-DAY / DÍA CERO
Vulnerabilidad desconocida para la que no existe parche. El más peligroso porque no hay defensa previa posible.
CVE (Common Vulnerabilities and Exposures)
Sistema estándar de identificación de vulnerabilidades. Ej: CVE-2025-53770 identifica la falla de SharePoint.
RCE (Remote Code Execution)
Ejecución remota de código. Permite al atacante correr comandos en el servidor víctima sin acceso físico.
MAN-IN-THE-MIDDLE (MitM)
Ataque donde el hacker se interpone en la comunicación entre dos partes. Permite leer, modificar o inyectar mensajes.
SHAREPOINT
Plataforma de Microsoft para compartir y gestionar documentos. Tiene versión local (on-premise) y en nube.
ON-PREMISE vs NUBE
On-premise: servidor físico en la organización. Nube (Cloud): servicio hospedado por el proveedor (ej: Microsoft 365).
2FA / MFA
Autenticación de 2 o múltiples factores. Agrega una capa de seguridad extra: algo que sabes + algo que tienes o eres.
CIFRADO E2E
Cifrado de extremo a extremo. Solo el emisor y receptor pueden leer el mensaje. Ni siquiera el proveedor tiene acceso.
IDS / IPS
Sistema de Detección/Prevención de Intrusiones. Monitorea tráfico en tiempo real y alerta o bloquea ataques.
SIEM
Security Information and Event Management. Centraliza logs de eventos del sistema para detectar patrones sospechosos.
EXPLOIT
Código o técnica que aprovecha una vulnerabilidad de software para ejecutar acciones no autorizadas.
WIPER
Malware diseñado para borrar/destruir datos permanentemente. Diferente del ransomware que los cifra para pedir rescate.
CISA
Agencia de Ciberseguridad e Infraestructura del Departamento de Seguridad Nacional de EE.UU.
DESERIALIZACIÓN
Proceso de convertir datos serializados de vuelta a objetos. Una falla aquí puede permitir ejecución de código malicioso.
